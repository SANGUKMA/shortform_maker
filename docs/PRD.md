# PRD: 롱폼-숏폼 자동 변환 서비스 (ClipForge)

> **문서 버전**: 1.0
> **작성일**: 2026-03-28
> **상태**: Draft

---

## 1. 개요

### 1.1 제품 요약

롱폼 영상 또는 YouTube 링크를 입력하면 AI가 하이라이트 구간을 자동 추출하고, 9:16 세로 숏폼 템플릿(네이비 프레임 + 제목/기관명)을 적용하여 여러 편의 숏폼 클립을 생성한다. 사용자는 생성된 클립 중 원하는 것을 선택하여 YouTube에 직접 업로드하거나 다운로드할 수 있다.

### 1.2 목표

- 롱폼 영상 1개에서 복수의 숏폼 클립을 자동 생성
- 편집 시간 90% 이상 절감
- 클립 생성부터 YouTube 업로드까지 원스톱 처리

### 1.3 타겟 사용자

- 기관/단체 홍보 담당자 (세미나, 강연, 회의 영상 → 숏폼 재가공)
- 1인 크리에이터 (롱폼 콘텐츠 → 쇼츠 확장)
- 미디어/뉴스 채널 운영자

## 2. 기능 요구사항

### 2.1 핵심 기능 (MVP)

#### F1. 영상 입력
| 항목 | 상세 |
|------|------|
| YouTube 링크 입력 | URL 붙여넣기 → 자동 다운로드 (yt-dlp) |
| 파일 업로드 | MP4, MOV, WEBM, MKV 지원, 최대 5GB |
| 메타데이터 표시 | 입력 즉시 제목, 길이, 해상도 미리보기 |

#### F2. 사용자 설정 입력
| 항목 | 상세 |
|------|------|
| 기관/채널명 | 하단 텍스트로 표시될 이름 |
| 클립 길이 선택 | 쇼츠(~60초) / 미드폼(1~6분) |
| 클립 개수 | 자동(AI 판단) 또는 사용자 지정(1~10개) |
| 자막 포함 여부 | ON/OFF 토글 |

#### F3. AI 하이라이트 분석
| 항목 | 상세 |
|------|------|
| 분석 엔진 | Gemini 2.0 Flash (영상 직접 입력) |
| 분석 대상 | 시각 + 오디오 + 텍스트 동시 분석 |
| 출력 | 하이라이트 구간 목록 (시작/종료 타임스탬프, 제목, 요약, 관심도 점수) |
| 처리 시간 | 영상 길이와 유사 (1시간 영상 → ~1~3분) |
| 비용 | ~$0.10/시간 (~130원) |

#### F4. 숏폼 클립 생성
| 항목 | 상세 |
|------|------|
| 클립 추출 | FFmpeg 타임스탬프 기반 정밀 컷 |
| 템플릿 적용 | 9:16 (1080x1920) 세로 프레임 |
| 배경색 | 진한 네이비 (#1B2A4A) |
| 상단 텍스트 | 클립별 제목 (AI 자동 생성, 편집 가능) |
| 하단 텍스트 | 기관/채널명 (사용자 입력) |
| 폰트 | Pretendard Bold (기본) / 맑은 고딕 Bold (폴백) |
| 영상 배치 | 원본 가로(16:9) 그대로 중앙 배치 |
| 자막 | Whisper 기반 자동 자막 burn-in (선택) |
| 인코딩 | H.264 + AAC, CRF 23, preset medium |

#### F5. 클립 미리보기 및 선택
| 항목 | 상세 |
|------|------|
| 카드형 목록 | 썸네일 + 제목 + 길이 + 관심도 점수 |
| 영상 재생 | 카드 내 인라인 미리보기 |
| 제목 편집 | 클릭하여 클립별 제목 수정 가능 |
| 다중 선택 | 체크박스로 복수 클립 선택 |

#### F6. 출력
| 항목 | 상세 |
|------|------|
| YouTube 업로드 | OAuth 2.0 인증 → 제목/설명/태그 설정 → 업로드 |
| 다운로드 | 개별 또는 일괄 ZIP 다운로드 |
| 업로드 설정 | 공개/비공개/미등록, 예약 공개 |

## 3. 기술 아키텍처

### 3.1 기술 스택

| 레이어 | 기술 | 선정 근거 |
|--------|------|----------|
| **UI** | NiceGUI | Python 통합, 카드 레이아웃, 실시간 진행률, FastAPI 내장 |
| **영상 다운로드** | yt-dlp | YouTube 다운로드 표준 도구, Python 라이브러리 사용 |
| **AI 분석** | Gemini 2.0 Flash API | 영상 직접 입력, 비용 최저($0.10/hr), 시각+오디오+텍스트 |
| **음성 전사** | Whisper API (OpenAI) | 자막 생성용, $0.006/분, 한국어 지원 |
| **영상 편집** | FFmpeg (subprocess) | 클립 컷, 템플릿 합성, 자막 burn-in, 텍스트 오버레이 |
| **YouTube 업로드** | YouTube Data API v3 | 공식 API, google-api-python-client |
| **언어** | Python 3.11+ | 전체 스택 통일 |

### 3.3 파이프라인 상세 흐름

```
사용자 입력 (URL or 파일)
    │
    ▼
[Step 1] Download ─────────────────────────────────────
    │  - YouTube URL → yt-dlp 다운로드 (1080p, mp4)
    │  - 파일 업로드 → 로컬 저장
    │  - 자동 자막 다운로드 (SRT)
    │  출력: source.mp4, source.ko.srt (있으면)
    ▼
[Step 2] Analyze ──────────────────────────────────────
    │  - Gemini 2.0 Flash에 영상 업로드
    │  - 프롬프트: 하이라이트 구간 추출 요청
    │  - 관심도 점수 + 제목 + 요약 포함
    │  출력: highlights.json
    │  [
    │    {
    │      "index": 1,
    │      "title": "클립 제목",
    │      "summary": "요약",
    │      "start": "00:02:15",
    │      "end": "00:03:10",
    │      "score": 92,
    │      "reason": "선정 이유"
    │    }, ...
    │  ]
    ▼
[Step 3] Extract ──────────────────────────────────────
    │  - highlights.json의 각 구간을 FFmpeg로 정밀 컷
    │  - 재인코딩 (키프레임 정밀도)
    │  출력: clips/clip_001.mp4, clip_002.mp4, ...
    ▼
[Step 4] Template ─────────────────────────────────────
    │  - 각 클립에 9:16 세로 템플릿 적용
    │  - FFmpeg: scale → pad → drawtext(상단) → drawtext(하단)
    │  - 배경: #1B2A4A, 상단: 제목, 하단: 기관명
    │  출력: shorts/short_001.mp4, short_002.mp4, ...
    ▼
[Step 5] Subtitle (선택) ─────────────────────────────
    │  - 기존 SRT가 있으면 구간에 맞게 분할
    │  - 없으면 Whisper API로 각 클립 전사
    │  - FFmpeg subtitles 필터로 burn-in
    │  출력: final/final_001.mp4, final_002.mp4, ...
    ▼
[사용자 선택] ─────────────────────────────────────────
    │  - 카드형 미리보기에서 클립 선택
    │  - 제목/설명 편집
    ▼
[Step 6] Output ───────────────────────────────────────
    ├─→ YouTube 업로드 (YouTube Data API v3)
    └─→ 다운로드 (개별 MP4 or ZIP)
```

## 4. 영상 템플릿 사양

### 4.1 레이아웃

```
┌──────────────────────────────┐
│        #1B2A4A 배경           │
│                              │  y=0
│    ┌──────────────────────┐  │
│    │   클립 제목 텍스트     │  │  y=60~120 (상단 텍스트 영역)
│    └──────────────────────┘  │
│                              │  y=120~360 (여백)
│  ┌──────────────────────────┐│
│  │                          ││
│  │                          ││
│  │     원본 영상 (16:9)      ││  y=360~968 (영상 영역, 1080x608)
│  │     1080 x 608            ││
│  │                          ││
│  │                          ││
│  └──────────────────────────┘│
│                              │  y=968~1800 (여백)
│    ┌──────────────────────┐  │
│    │   기관/채널 이름       │  │  y=1800~1860 (하단 텍스트 영역)
│    └──────────────────────┘  │
│                              │  y=1920
└──────────────────────────────┘
         1080 x 1920
```

### 4.2 스타일 상세

| 요소 | 속성 | 값 |
|------|------|-----|
| **캔버스** | 크기 | 1080 x 1920 (9:16) |
| **배경** | 색상 | #1B2A4A (진한 네이비) |
| **원본 영상** | 크기 | 1080 x 608 (16:9 비율 유지) |
| **원본 영상** | 위치 | 수직/수평 중앙 |
| **상단 제목** | 폰트 | Pretendard Bold |
| **상단 제목** | 크기 | 52~56px |
| **상단 제목** | 색상 | #FFFFFF (흰색) |
| **상단 제목** | 정렬 | 수평 중앙 |
| **상단 제목** | 위치 | y=80 |
| **상단 제목** | 외곽선 | 3px, #000000 |
| **하단 기관명** | 폰트 | Pretendard Bold |
| **하단 기관명** | 크기 | 36~40px |
| **하단 기관명** | 색상 | #CCCCCC (밝은 회색) |
| **하단 기관명** | 정렬 | 수평 중앙 |
| **하단 기관명** | 위치 | y = h-120 |
| **하단 기관명** | 외곽선 | 2px, #000000 |
| **자막** (선택) | 폰트 | Pretendard / 맑은 고딕 |
| **자막** | 크기 | 28px |
| **자막** | 위치 | 영상 영역 하단 (영상 내부) |
| **자막** | 외곽선 | 2px, 검정 |

### 4.3 FFmpeg 템플릿 명령어

```bash
ffmpeg -i clip.mp4 \
  -vf "\
    scale=1080:-2,\
    pad=1080:1920:(ow-iw)/2:(oh-ih)/2:color=#1B2A4A,\
    drawtext=fontfile='fonts/Pretendard-Bold.otf':\
      text='클립 제목':fontsize=54:fontcolor=white:\
      x=(w-text_w)/2:y=80:borderw=3:bordercolor=black,\
    drawtext=fontfile='fonts/Pretendard-Bold.otf':\
      text='기관명':fontsize=38:fontcolor=#CCCCCC:\
      x=(w-text_w)/2:y=h-120:borderw=2:bordercolor=black\
  " \
  -c:v libx264 -preset medium -crf 23 \
  -c:a aac -b:a 128k \
  output.mp4
```

### 5.1 필수 API 키

| 서비스 | 용도 | 환경변수 | 비용 |

### 6.2 사용자 흐름

1. 앱 접속 (`http://localhost:8080`)
2. YouTube URL 붙여넣기 또는 영상 파일 업로드
3. 기관명 입력, 길이/개수/자막 설정
4. "분석 시작" 클릭
5. 진행 상황 실시간 확인 (Step 1~5)
6. 완료 후 클립 카드 목록에서 미리보기
7. 원하는 클립 체크박스 선택
8. "YouTube 업로드" → 제목/설명/태그 편집 → 업로드
9. 또는 "다운로드" → ZIP 파일로 일괄 다운로드

## 7. 비기능 요구사항

### 7.1 성능

| 항목 | 목표 |
|------|------|
| 10분 영상 전체 처리 | 3분 이내 |
| 30분 영상 전체 처리 | 8분 이내 |
| 1시간 영상 전체 처리 | 15분 이내 |
| YouTube 업로드 | 클립 1개당 1분 이내 (100MB 기준) |

### 7.2 안정성

- 네트워크 오류 시 자동 재시도 (최대 3회)
- 파이프라인 중간 실패 시 체크포인트에서 재개 가능
- 대용량 파일(5GB) 처리 시 메모리 관리 (스트리밍 처리)

### 7.3 보안

- API 키는 .env 파일로 관리, 코드에 하드코딩 금지
- YouTube OAuth 토큰은 로컬 암호화 저장
- 업로드된 영상은 처리 완료 후 사용자 선택에 따라 삭제 가능
